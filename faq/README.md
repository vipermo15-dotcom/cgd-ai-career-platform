## 1. 플랫폼 사용

**Q1. CGD 플랫폼 URL이 어떻게 되나요?**
> https://cgdplatform-pm7dnqip.manus.space 입니다.
> 교수자에게 계정을 발급받은 후 사용할 수 있습니다.

**Q2. 비밀번호를 잊어버렸어요.**
> 로그인 페이지의 "비밀번호 찾기" 클릭 후 가입 이메일로 재설정 링크를 받으세요.
> 이메일이 오지 않으면 교수자에게 문의하세요.

**Q3. 플랫폼에서 AI 에이전트가 응답하지 않아요.**
> 1. 인터넷 연결 확인
> 2. 브라우저 새로 고침 (F5)
> 3. 다른 브라우저 시도
> 4. 그래도 안 되면 교수자에게 문의 (일시적 서버 오류일 수 있음)

**Q4. 플랫폼에 파일을 업로드하는 방법은?**
> 각 에이전트의 "파일 첨부" 아이콘을 클릭하여 업로드합니다.
> 지원 형식: PDF, JPG, PNG, DOC, DOCX (각 10MB 이하)

**Q5. 진로지도 카드는 어떻게 작성하나요?**
> 내 프로필 > 진로지도 카드 메뉴에서 작성합니다.
> 6개 항목(희망 직군, 보유 스킬, 보완 필요 스킬, 목표 기업 유형, 취업 시기, 특이사항)을 입력하면 됩니다.

## 2. Git / GitHub

**Q6. Git 설치 후 `git --version`이 안 돼요.**
> Windows: 설치 후 터미널을 완전히 닫고 다시 열어보세요.
> Mac: 터미널에서 `xcode-select --install` 실행 후 재시도.

**Q7. `git push` 시 "Authentication failed" 오류가 나요.**
> GitHub Personal Access Token(PAT)을 발급받아야 합니다.
> GitHub > Settings > Developer Settings > Tokens에서 생성하세요.
> 비밀번호 입력란에 PAT를 입력하면 됩니다.

**Q8. 실수로 커밋한 파일을 되돌리고 싶어요.**
> ```bash
> git revert HEAD    # 마지막 커밋 되돌리기 (기록은 남음)
> ```
> 절대 `git reset --hard`는 혼자 쓰지 마세요! (데이터 삭제 위험)

**Q9. .env 파일이 GitHub에 올라갔어요.**
> 즉시 해당 API Key를 교체하세요! (노출된 키는 해킹 위험)
> 1. Firecrawl/Anthropic에서 새 키 발급
> 2. `.gitignore`에 `.env` 추가
> 3. `git rm --cached .env` 실행
> 4. 새 커밋으로 .env 파일 제거

**Q10. GitHub 저장소를 Private으로 하면 포트폴리오 용도로 쓸 수 없나요?**
> Private 저장소는 다른 사람이 볼 수 없습니다.
> 포트폴리오용은 Public으로 설정하고, API Key 등은 절대 올리지 마세요.

## 3. Claude AI 활용

**Q11. 무료 플랜에서 하루 몇 번 사용할 수 있나요?**
> 약 10~20회 대화 가능합니다. (Claude의 정책에 따라 변동)
> 한 번의 긴 대화가 여러 번의 짧은 대화보다 효율적입니다.

**Q12. Claude가 틀린 정보를 알려줬어요.**
> Claude는 실수할 수 있습니다. 중요한 정보(기업 연봉, 마감일 등)는 반드시 직접 확인하세요.
> "이 정보를 확인해줘" 보다는 "이 정보의 출처를 알려줘"라고 물어보세요.

**Q13. Claude가 영어로 답변해요.**
> 프롬프트 마지막에 "한국어로 답변해주세요"를 추가하거나
> Project Instructions에 "항상 한국어로 답변"을 설정하세요.

**Q14. Claude Projects는 무엇인가요?**
> AI에게 나의 맥락을 미리 알려두는 공간입니다.
> "나는 CGD 전공 학생이고..." 같은 배경 정보를 저장해두면
> 매번 설명하지 않아도 됩니다.

**Q15. 프롬프트를 어떻게 하면 잘 쓸 수 있나요?**
> 핵심 5가지:
> 1. 역할 부여: "당신은 ~~입니다"
> 2. 맥락 제공: 내가 누구인지, 상황이 무엇인지
> 3. 구체적 지시: "3가지", "500자" 등 수치 포함
> 4. 형식 지정: "표로", "번호 목록으로"
> 5. 예시 제공: "이런 식으로 작성해주세요"

## 4. MCP 설치

**Q16. Claude Desktop이 뭔가요?**
> 브라우저가 아닌 컴퓨터에 설치하는 Claude 앱입니다.
> MCP를 통해 GitHub, 로컬 파일, Firecrawl 등과 연동할 수 있습니다.

**Q17. MCP 서버 설정 후 재시작해도 연결이 안 돼요.**
> 1. JSON 문법 오류 확인 (쉼표, 괄호 체크)
> 2. Node.js 버전 확인 (v18 이상)
> 3. Claude Desktop 완전 종료 후 재시작
> 4. 설정 파일 경로가 맞는지 확인

**Q18. MCP 설정 파일 위치는 어디인가요?**
> - Mac: `~/Library/Application Support/Claude/claude_desktop_config.json`
> - Windows: `%APPDATA%\Claude\claude_desktop_config.json`

**Q19. "npx not found" 오류가 나요.**
> Node.js가 설치되지 않은 것입니다.
> https://nodejs.org 에서 LTS 버전을 설치하세요.

**Q20. PAT(Personal Access Token)를 어디서 발급받나요?**
> GitHub > 오른쪽 상단 프로필 클릭 > Settings > Developer settings > Personal access tokens > Tokens (classic) > Generate new token

## 5. Firecrawl

**Q21. Firecrawl 무료 플랜에서 몇 크레딧을 받나요?**
> 월 500 크레딧입니다. 스크래핑 1회 = 1 크레딧.

**Q22. "Rate limit exceeded" 오류가 자주 나요.**
> 요청 간에 `time.sleep(2)` 같은 딜레이를 추가하세요.
> 또는 API 호출 횟수를 줄여보세요.

**Q23. Firecrawl로 특정 사이트가 수집이 안 돼요.**
> 일부 사이트는 봇 차단이 있어 수집이 어렵습니다.
> `waitFor` 값을 늘려보세요: `'waitFor': 5000` (5초 대기)

**Q24. 수집된 Markdown에 이상한 문자가 많아요.**
> `onlyMainContent: True` 옵션을 추가하면 광고/메뉴 등이 제거됩니다.

**Q25. .env 파일에 API Key를 저장했는데 인식이 안 돼요.**
> 코드에서 `from dotenv import load_dotenv` 후 `load_dotenv()`를 먼저 호출해야 합니다.

## 6. 포트폴리오 제작

**Q26. 포트폴리오 페이지는 몇 장이 적당한가요?**
> 20~30페이지가 적당합니다.
> 10페이지 이하는 너무 적고, 40페이지 이상은 보는 사람이 지칩니다.

**Q27. 포트폴리오에 어떤 작업을 넣어야 하나요?**
> - 완성도 높은 것 (미완성 제외)
> - 지원 직군과 관련 있는 것
> - 최근 1~2년 작업
> - 스스로 설명할 수 있는 것

**Q28. 학교 과제도 포트폴리오에 넣어도 되나요?**
> 네, 가능합니다. 단, 과제임을 솔직히 밝히고
> 과제를 완성하는 과정에서 무엇을 배웠는지 설명을 추가하세요.

**Q29. Behance가 뭔가요? 꼭 올려야 하나요?**
> 디자이너 포트폴리오 공유 플랫폼입니다.
> 필수는 아니지만, URL로 공유할 수 있어 이메일 제출에 유리합니다.
> 채용 담당자가 검색으로 찾을 수도 있습니다.

**Q30. PDF 파일 크기가 너무 커요.**
> Adobe Acrobat의 "PDF 크기 줄이기" 기능을 사용하거나
> ilovepdf.com에서 압축하세요. 목표: 20MB 이하

## 7. 자기소개서

**Q31. 자기소개서를 Claude로 쓰면 표나요?**
> Claude는 초안만 사용하고 반드시 자신의 경험과 목소리로 수정해야 합니다.
> AI 글 특유의 패턴이 있어 경험 많은 채용 담당자는 알 수 있습니다.

**Q32. 자기소개서 분량이 얼마나 돼야 하나요?**
> 기업마다 다르지만 일반적으로:
> - 항목별 작성: 200~500자
> - 통합 작성: 1,000~2,000자
> 정해진 분량보다 5~10% 적게 쓰는 것이 보기 좋습니다.

**Q33. "열정", "최선", "노력" 같은 단어를 쓰면 안 되나요?**
> 사용 자체는 가능하지만, 이런 단어는 증거가 필요합니다.
> "열정이 있습니다" 보다 "3개월간 매주 10시간 이상 작업했습니다"가 더 설득력 있습니다.

**Q34. 지원 동기를 어떻게 쓰면 좋을까요?**
> "이 제품/서비스를 직접 사용하다가..." 처럼 구체적인 계기를 쓰세요.
> "어릴 때부터 디자인에 관심이 있었습니다"는 피하세요.

**Q35. 수상 경력이 없어도 취업이 가능한가요?**
> 충분히 가능합니다. 포트폴리오 퀄리티가 수상보다 중요합니다.
> 대신 이렇게 어필하세요: "수상 대신 실제 프로젝트로 역량을 증명합니다"

## 8. 기업분석

**Q36. 기업을 분석할 때 어디서 정보를 얻나요?**
> 1. 기업 공식 홈페이지 (채용 페이지 특히)
> 2. 뉴스 (네이버, 구글 뉴스)
> 3. 블라인드 (직원 리뷰)
> 4. LinkedIn (기업 페이지)
> 5. Firecrawl + Claude (자동 분석)

**Q37. 면접 전날 기업 리서치를 얼마나 해야 하나요?**
> 최소 30분, 이상적으로는 1~2시간입니다.
> 알아야 할 것: 기업 3줄 요약, 최근 이슈 2가지, 지원 직군 주요 업무

**Q38. 모르는 기업에 지원해도 되나요?**
> 가능하지만, 자기소개서와 면접 준비가 더 어렵습니다.
> 지원 전 최소 30분 이상 기업 조사 필수.

**Q39. Firecrawl로 기업 정보 수집 시 법적 문제가 없나요?**
> robots.txt를 준수하고, 개인정보가 아닌 공개 정보만 수집하면 됩니다.
> 수집한 정보는 개인 학습용으로만 사용하세요.

**Q40. 채용공고에 연봉이 "회사 내규에 따름"이라고만 나와 있어요.**
> 일반적으로 협상 가능하다는 신호입니다.
> 면접 합격 후 처우 협의 단계에서 희망 연봉을 말할 수 있습니다.

## 9. 면접 준비

**Q41. 1분 자기소개에서 무조건 해야 할 말은?**
> 이름 + 핵심 강점 1~2가지 + 지원 직군 관련 경험 1가지.
> 길게 말하는 것보다 핵심을 간결하게!

**Q42. 포트폴리오 설명 시간은 얼마나 줘야 하나요?**
> 보통 3~5분입니다. 면접관이 "좀 더 설명해주세요"라고 하면 추가 설명 가능.

**Q43. 면접에서 모르는 질문이 나오면 어떻게 하나요?**
> "잠깐 생각해봐도 될까요?" (5~10초)
> 모른다면: "아직 그 부분은 경험이 없지만, 이렇게 접근할 것 같습니다..."

**Q44. "당신의 약점이 뭔가요?" 질문에 어떻게 답하나요?**
> 실제 약점을 말하되, 극복 과정을 함께 이야기하세요.
> 직무에 치명적인 약점은 피하세요.

**Q45. 면접 복장은 어떻게 하면 좋나요?**
> 기업 분위기에 따라 다르지만, 기본은 비즈니스 캐주얼입니다.
> 스타트업: 깔끔한 캐주얼 OK
> 대기업/공기업: 정장 또는 단정한 비즈니스 캐주얼
> 에이전시/크리에이티브: 자신의 스타일을 약간 반영해도 OK

## 10. 취업 일반

**Q46. 신입이 연봉 협상을 해도 되나요?**
> 네, 당연히 가능합니다. 합격 통보 후 처우 협의 단계에서 정중하게 요청하세요.
> "희망 연봉을 말씀드려도 될까요?"라고 먼저 물어보세요.

**Q47. 공채와 수시채용 중 어떤 것이 유리한가요?**
> CGD 같은 전문 직군은 수시채용이 많습니다.
> 원티드, 사람인, 잡코리아 수시채용을 상시 모니터링하세요.

**Q48. 취업 준비 기간이 얼마나 걸리나요?**
> 평균 3~6개월입니다. 포트폴리오 완성도와 직군에 따라 다릅니다.
> 서두르지 말고 포트폴리오가 진짜 준비됐을 때 지원하세요.

**Q49. 불합격 후 같은 기업에 다시 지원해도 되나요?**
> 대부분 6개월~1년 후 가능합니다. (기업마다 정책 다름)
> 재지원 시 달라진 점을 명확히 어필해야 합니다.

**Q50. 취업 후 포트폴리오 관리는 어떻게 하나요?**
> 입사 후에도 사이드 프로젝트나 회사 작업(허용된 범위 내)으로 포트폴리오를 계속 업데이트하세요.
> Behance와 GitHub 주기적으로 업데이트를 권장합니다.

**Q51. 교수자에게 추천서를 받을 수 있나요?**
> 네, 교수자와 상담 후 요청하면 됩니다. (template/취업추천서.md 양식 활용)
> 추천서는 보통 1~2주 전에 미리 요청하세요.

**Q52. 졸업 전에 취업이 되면 어떻게 하나요?**
> 축하드립니다! 교수자에게 알리고 입사 일정을 조율하세요.
> 졸업 전 취업은 학교와 협의하여 조기 수료 또는 휴학 후 입사가 가능합니다.

**Q53. AI가 디자이너를 대체할까요?**
> AI는 도구입니다. AI를 잘 활용하는 디자이너가 경쟁력을 갖습니다.
> 이 과정에서 배운 Claude, Firecrawl, Midjourney 활용 능력이 바로 그 경쟁력입니다.

**Q54. 포트폴리오가 완성되지 않았는데 지원해도 되나요?**
> 권장하지 않습니다. 불완성 포트폴리오는 오히려 마이너스가 될 수 있습니다.
> 포트폴리오 완성 후 지원하되, 지원 기간을 너무 늦추지는 마세요.

**Q55. 어떤 회사부터 지원해야 하나요?**
> 1순위: 정말 가고 싶은 회사 (꿈의 회사)
> 2순위: 준비가 충분히 된 회사
> 3순위: 경험 삼아 지원해볼 회사
> 이 순서로 지원하되 모두 동시에 진행하는 것을 권장합니다.
