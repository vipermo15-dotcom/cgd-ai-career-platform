## 1. Git/GitHub 문제

### 문제 1: `git push` 시 인증 오류

**증상**:
```
remote: Support for password authentication was removed on August 13, 2021.
fatal: Authentication failed for 'https://github.com/...'
```

**원인**: GitHub는 2021년부터 비밀번호 인증을 폐지하고 Personal Access Token(PAT) 사용 필요

**해결방법**:
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. "Generate new token (classic)" 클릭
3. 권한: `repo` 전체 체크
4. 생성된 토큰 복사
5. `git push` 시 비밀번호 대신 토큰 입력

또는 GitHub CLI 사용:
```bash
# GitHub CLI 설치 후
gh auth login
# 브라우저 로그인 방식 선택
```

---

### 문제 2: `Permission denied (publickey)`

**증상**:
```
Permission denied (publickey).
fatal: Could not read from remote repository.
```

**해결방법**: HTTPS 방식으로 변경
```bash
# 현재 원격 URL 확인
git remote -v

# SSH에서 HTTPS로 변경
git remote set-url origin https://github.com/사용자명/저장소명.git
```

---

### 문제 3: 브랜치 이름이 `master`인데 `main`으로 바꾸고 싶어요

```bash
# 현재 브랜치 이름을 main으로 변경
git branch -m master main

# 원격에 main 브랜치 Push
git push -u origin main

# 기존 master 삭제
git push origin --delete master
```

---

### 문제 4: 커밋을 잘못했어요 (아직 Push 전)

```bash
# 마지막 커밋 메시지만 수정
git commit --amend -m "수정된 커밋 메시지"

# 마지막 커밋 취소 (파일 변경사항은 유지)
git reset HEAD~1

# 마지막 커밋 완전 취소 (파일 변경사항도 취소)
git reset --hard HEAD~1
```

---

### 문제 5: 충돌(Conflict) 발생

```
CONFLICT (content): Merge conflict in 파일명.md
Automatic merge failed; fix conflicts and then commit the result.
```

**해결방법**:
1. VSCode에서 충돌 파일 열기
2. 충돌 표시 찾기:
```
<<<<<<< HEAD
내 변경사항
=======
원격의 변경사항
>>>>>>> origin/main
```
3. 원하는 내용으로 수동 수정 (충돌 표시 삭제)
4. 저장 후 커밋:
```bash
git add 파일명.md
git commit -m "fix: 충돌 해결"
```

## 2. VSCode 문제

### 문제 1: 한국어 깨짐 (인코딩 오류)

**증상**: 파일을 열면 글자가 깨져 보임 (예: ȼȦȭ)

**해결방법**:
1. 우하단 인코딩 표시 클릭 (UTF-8 또는 EUC-KR)
2. "Reopen with Encoding" 선택
3. "Korean (EUC-KR)" 또는 "UTF-8" 선택

파일 저장 시 UTF-8로 통일:
1. 설정(`Ctrl+,`) → "files.encoding" 검색
2. "utf8" 선택

---

### 문제 2: Git 기능이 VSCode에서 안 보여요

**해결방법**:
1. Git이 설치되어 있는지 확인: `git --version`
2. VSCode 재시작
3. 설정(`Ctrl+,`) → "git.enabled" 검색 → 활성화

---

### 문제 3: 터미널에서 `code` 명령어가 안 돼요 (macOS)

**해결방법**:
1. VSCode 실행
2. `Cmd+Shift+P` → "Shell Command: Install 'code' command in PATH"
3. 터미널 재시작

## 3. Claude 문제

### 문제 1: 사용량 한도 초과

**증상**: "사용량 한도에 도달했습니다. X시간 후에 다시 시도해주세요"

**해결방법**:
- 1-3시간 기다리면 사용량 리셋
- 기다리는 동안 Gemini 또는 ChatGPT 사용
- 반복적으로 한도 도달 시 Claude Pro 업그레이드 고려

---

### 문제 2: 파일 첨부가 안 돼요

**해결방법**:
- 지원 파일 형식 확인: PDF, Word, 텍스트, 이미지
- 파일 크기 확인: 10MB 이하 권장
- 파일명에 특수문자 제거 후 재시도

파일 내용을 복사해서 붙여넣는 방식으로 대체 가능:
```
다음은 내 이력서 내용이야:
---
[이력서 내용 붙여넣기]
---
위 이력서를 분석해줘.
```

---

### 문제 3: 답변이 중간에 끊겨요

**해결방법**:
- "계속해줘" 또는 "이어서 작성해줘" 입력
- 긴 글은 분량을 나눠서 요청:
  ```
  자기소개서를 한 번에 쓰지 말고
  단락별로 나눠서 하나씩 작성해줘.
  먼저 성장과정 단락부터.
  ```

## 4. MCP 문제

### 문제 1: MCP 서버가 연결 안 됨

**확인 순서**:

1. 설정 파일 JSON 문법 오류 확인:
```bash
# Windows
node -e "JSON.parse(require('fs').readFileSync('%APPDATA%\\Claude\\claude_desktop_config.json', 'utf8'))"

# macOS
node -e "JSON.parse(require('fs').readFileSync(process.env.HOME + '/Library/Application Support/Claude/claude_desktop_config.json', 'utf8'))"
```

2. Node.js 설치 확인:
```bash
node --version
npm --version
```

3. Claude Desktop 완전 종료 후 재시작
   - macOS: `Cmd+Q` 후 재실행
   - Windows: 작업 표시줄 아이콘 우클릭 → 종료

---

### 문제 2: GitHub MCP 오류

**증상**: `GitHubError: 401 Unauthorized`

**해결방법**:
1. Personal Access Token 만료 여부 확인
2. 새 토큰 발급 후 설정 파일 업데이트
3. Claude Desktop 재시작

## 5. Firecrawl 문제

### 문제 1: 스크래핑 결과가 비어있어요

**원인과 해결**:
- 동적 JavaScript 페이지: `waitFor` 옵션 추가
```python
result = app.scrape_url(url, params={
    'formats': ['markdown'],
    'waitFor': 2000  # 2초 기다리기
})
```

- 봇 차단: `stealth` 옵션 시도 (Firecrawl 고급 기능)

---

### 문제 2: API 사용량 초과

**해결방법**:
- 교수자에게 추가 사용량 요청
- 불필요한 스크래핑 요청 최소화
- 한 번 수집한 결과를 파일로 저장하여 재사용

## 6. 일반적인 문제

### 문제: 인터넷이 느려요

- Claude, Gemini: 텍스트 기반이므로 느린 인터넷에서도 사용 가능
- 큰 파일 업로드 시 WiFi 환경 권장

### 문제: 윈도우 경로에서 한글이 포함된 경우 오류

Git Bash에서 한글 경로 문제:
```bash
# 설정에서 한글 경로 지원
git config --global core.quotepath false
git config --global i18n.commitEncoding UTF-8
git config --global i18n.logOutputEncoding UTF-8
```

한글 폴더명 대신 영문 폴더명 사용 권장:
- ❌ `C:\사용자\문서\포트폴리오`
- ✅ `C:\Users\username\portfolio`
