## 1. 좋은 프롬프트의 원칙

### 5가지 핵심 원칙

**원칙 1: 명확성 (Clarity)**
- 원하는 것을 명확하게 명시
- 애매한 표현 피하기
- 구체적인 단어 사용

```
❌ 나쁜 예: "자기소개서 써줘"
✅ 좋은 예: "그래픽 디자이너 포지션 지원용 자기소개서를 1000자로 작성해줘"
```

**원칙 2: 맥락 제공 (Context)**
- 배경 정보 제공
- 목적과 용도 설명
- 관련 자료 첨부

```
❌ 나쁜 예: "면접 질문 알려줘"
✅ 좋은 예: "출판사 편집 디자이너 1차 면접에서 자주 나오는 질문 10가지를 알려줘. 
              나는 Adobe InDesign 2년, Photoshop 3년 경험이 있어."
```

**원칙 3: 형식 지정 (Format)**
- 원하는 출력 형식 명시
- 분량 지정
- 구조 요청

```
다음 형식으로 작성해줘:
- 분량: 800-1000자
- 구성: 첫째 단락(관심 계기) + 둘째 단락(역량 어필) + 셋째 단락(포부)
- 어조: 진솔하고 자신감 있게
- 금지: 상투적 표현, 과장
```

**원칙 4: 예시 제공 (Examples)**
- 원하는 스타일의 예시 제공
- 참고 샘플 첨부
- 좋은/나쁜 예시 함께 제시

**원칙 5: 반복 개선 (Iteration)**
- 한 번에 완벽한 결과 기대하지 않기
- 피드백을 주고 개선 요청
- 여러 버전 비교

## 2. 역할 부여 (Role Prompting)

역할 부여는 AI에게 특정 전문가의 관점으로 답변하도록 하는 기법입니다.

### 기본 역할 부여 구조

```
너는 [역할]이야.
[역할 특성 및 경험 설명]
[요청 사항]
```

### 취업 준비에 유용한 역할들

**1. 채용 담당자**
```
너는 대형 광고 대행사의 크리에이티브 팀 채용 담당자야.
디자이너 채용 경험이 10년 이상이고, 
포트폴리오와 자기소개서를 매일 100개 이상 검토해.
내 자기소개서를 보고 솔직한 피드백을 줘.
합격 가능성도 0-100% 사이로 평가해줘.
```

**2. 현직 디자이너**
```
너는 카카오에서 5년째 일하는 브랜드 디자이너야.
신입 디자이너 멘토링을 한 경험도 있어.
내 포트폴리오를 보고 현업 관점에서 피드백을 줘.
특히 신입이 놓치기 쉬운 부분을 중심으로 말해줘.
```

**3. 면접관**
```
너는 중소 디자인 에이전시 대표이자 면접관이야.
신입 디자이너를 자주 채용하며, 실력보다 태도와 잠재력을 중시해.
지금부터 나와 1대1 면접을 진행해줘.
면접 후 총평과 합격 여부도 알려줘.
```

**4. 커리어 코치**
```
너는 디자인 분야 취업 전문 커리어 코치야.
매년 50명 이상의 디자이너 취업을 성공시켰어.
내 현재 상황을 분석하고 90일 취업 전략을 세워줘.
```

## 3. 단계별 사고 (Chain of Thought)

복잡한 작업을 단계별로 분해하여 더 좋은 결과를 얻는 기법입니다.

### 기본 구조

```
이 문제를 단계별로 생각해줘:
1단계: [첫 번째 분석]
2단계: [두 번째 분석]
3단계: [최종 결론]
```

### 자기소개서 작성 예시

```
자기소개서를 작성하기 전에 다음 단계를 거쳐줘:

1단계 분석: 채용공고에서 핵심 키워드 10개 추출
2단계 매칭: 내 경험 중 각 키워드와 연결되는 것 찾기
3단계 스토리: 가장 강력한 에피소드 2개 선택
4단계 초안: 위 분석을 바탕으로 자기소개서 초안 작성
5단계 검토: 초안에서 진부한 표현, 과장 찾아 수정

각 단계의 결과를 보여준 후 다음 단계로 진행해줘.
```

### 기업 분석 Chain of Thought

```
[기업 홈페이지 내용 첨부]

위 내용을 다음 순서로 분석해줘:

STEP 1: 회사의 주요 사업과 수익 모델 파악
STEP 2: 디자인팀이 하는 구체적인 업무 추론
STEP 3: 회사가 원하는 디자이너 역량 분석
STEP 4: 내가 이 회사의 면접에서 강조해야 할 포인트
STEP 5: 이 회사 지원을 위한 포트폴리오 수정 제안
```

## 4. 취업 관련 프롬프트 20가지

### 자기소개서 관련

**프롬프트 1: 지원동기 작성**
```
나는 [회사명]의 [직무]에 지원하려고 해.
다음 정보를 바탕으로 지원동기 800자를 작성해줘.

회사 정보: [홈페이지에서 수집한 내용]
내 경험: [관련 경험 목록]
핵심 강점: [강점 2-3가지]

조건: 회사의 미션과 내 역량이 연결되도록, 구체적인 에피소드 포함
```

**프롬프트 2: 자기소개서 첨삭**
```
아래 자기소개서를 첨삭해줘.

[자기소개서 전문]

첨삭 기준:
- 진부한 표현 표시 및 대안 제시
- 구체성이 부족한 부분 지적
- 채용 담당자 관점에서 매력도 평가 (1-10점)
- 추가하면 좋을 내용 제안
```

**프롬프트 3: 성장과정 작성**
```
너는 자기소개서 전문 컨설턴트야.
아래 정보를 바탕으로 성장과정 500자를 작성해줘.

디자이너가 되고 싶었던 계기: [계기 설명]
디자인에서 가장 의미 있었던 경험: [경험]
성장 과정에서 극복한 어려움: [어려움과 극복 방법]
```

**프롬프트 4: 입사 후 포부**
```
[회사명]에 합격했다고 가정하고,
입사 후 3년간의 성장 계획을 담은 입사 후 포부 500자를 작성해줘.

회사 사업 분야: [분야]
내가 기여하고 싶은 부분: [기여 방향]
장기 목표: [목표]
```

**프롬프트 5: 자기소개서 버전 비교**
```
같은 내용으로 두 가지 스타일의 자기소개서를 작성해줘.

Version A: 성과 중심 (수치, 결과 강조)
Version B: 과정 중심 (경험, 배움 강조)

어떤 버전이 더 효과적인지 이유와 함께 추천해줘.
```

### 기업 분석 관련

**프롬프트 6: 기업 분석 요약**
```
다음 기업 정보를 분석해서 면접 준비용 요약 정리를 해줘.

[기업 홈페이지 내용 붙여넣기]

정리 형식:
1. 핵심 비즈니스 (3문장)
2. 기업 문화 키워드 (5개)
3. 최근 주요 이슈 (있다면)
4. 면접에서 활용할 스토리텔링 포인트
```

**프롬프트 7: 경쟁사 비교 분석**
```
[회사명]과 주요 경쟁사를 비교 분석해줘.

주요 비교 항목:
- 사업 방향과 전략 차이
- 디자인 스타일 특징
- 기업 문화 차이
- 내가 이 회사를 선택한 이유로 어필할 수 있는 차별점
```

**프롬프트 8: 채용공고 키워드 추출**
```
아래 채용공고에서 중요도 순으로 키워드를 추출해줘.

[채용공고 전문]

추출 기준:
- 자격요건에서 반복되는 단어
- 우대사항의 핵심 역량
- 업무 설명에서 자주 나오는 동작 표현
- 자기소개서에 반드시 포함해야 할 단어 TOP 10
```

**프롬프트 9: 합격 가능성 분석**
```
아래 채용공고와 내 프로필을 비교하여 합격 가능성을 분석해줘.

[채용공고 내용]

내 프로필:
- 학력/교육: [교육 배경]
- 경력/경험: [경험]
- 보유 기술: [스킬]
- 포트폴리오: [주요 작업]

항목별 매칭 점수 (0-10점)와 총평, 개선 전략도 알려줘.
```

**프롬프트 10: 지원 순위 정하기**
```
아래 3개 기업 중 어디에 먼저 지원할지 전략적으로 추천해줘.

기업 A: [기업명, 직무, 공고 마감일]
기업 B: [기업명, 직무, 공고 마감일]
기업 C: [기업명, 직무, 공고 마감일]

내 상황: [현재 준비 상황]
고려 요소: 합격 가능성, 취업 조건, 커리어 성장
```

### 면접 준비 관련

**프롬프트 11: 모의 면접 시작**
```
지금부터 나와 [회사명] [직무] 1차 면접을 진행해줘.
너는 면접관이고, 나는 지원자야.

면접 방식: 10개 질문, 각 답변 후 꼬리 질문 1개
평가 기준: 직무 이해도, 포트폴리오 설명력, 커뮤니케이션
마지막에 총점(100점)과 합격 여부 판단해줘.

시작!
```

**프롬프트 12: 면접 답변 개선**
```
내 면접 답변을 개선해줘.

질문: "왜 디자이너가 되고 싶으셨나요?"
내 답변: [현재 답변]

개선 기준:
- STAR 기법 적용 (상황-과제-행동-결과)
- 지나치게 길지 않게 (2분 이내)
- 진정성이 느껴지도록
- 개선된 버전 2가지 제시
```

**프롬프트 13: 포트폴리오 발표 스크립트**
```
포트폴리오 발표를 위한 3분 스크립트를 작성해줘.

발표할 작업물:
1. [프로젝트명]: [간단한 설명]
2. [프로젝트명]: [간단한 설명]

포함할 내용:
- 각 프로젝트의 목적과 과제
- 내가 맡은 역할
- 디자인 결정의 이유
- 배운 점과 성장
```

**프롬프트 14: 취약점 질문 대비**
```
내 취약점에 관한 면접 질문 대비를 도와줘.

내 취약점:
- 경력 없음 (교육 수료 예정)
- [특정 기술] 미숙
- [기타 약점]

각 취약점에 대한 긍정적 답변 전략을 알려줘.
약점을 솔직하게 인정하되 극복 의지를 보여주는 방식으로.
```

**프롬프트 15: 역질문 준비**
```
면접 마지막에 "질문 있으신가요?" 할 때 할 수 있는
좋은 역질문 10가지를 알려줘.

[회사명] [직무] 관련 역질문
- 회사 문화 파악용
- 성장 기회 확인용
- 팀 분위기 파악용
각 질문의 의도와 좋은 인상을 주는 이유도 설명해줘.
```

### 포트폴리오 관련

**프롬프트 16: 포트폴리오 구성 전략**
```
내 포트폴리오를 [회사명] [직무] 지원에 맞게 구성해줘.

현재 작업물 목록:
1. [작업 1]: [간단한 설명]
2. [작업 2]: [간단한 설명]
3. [작업 3]: [간단한 설명]
4. [작업 4]: [간단한 설명]
5. [작업 5]: [간단한 설명]

선택 기준: 직무 관련성, 완성도, 다양성
최적의 순서와 제외할 작업도 추천해줘.
```

**프롬프트 17: 작업물 설명 문구 작성**
```
포트폴리오의 각 작업물 설명 문구를 영어로 작성해줘.

작업물: [작업 이름]
목적: [제작 이유]
내 역할: [담당 범위]
사용 툴: [Adobe CC 등]
결과: [성과나 배운 점]

100단어 이내의 영어 설명문을 작성해줘.
```

**프롬프트 18: 포트폴리오 피드백 요청**
```
[이미지 첨부]

이 포트폴리오 페이지에 대해 전문가 피드백을 줘.

평가 기준:
1. 레이아웃 구성 (가독성)
2. 타이포그래피 선택
3. 컬러 사용
4. 전반적인 완성도
5. 채용 담당자 입장에서의 첫인상

점수 (1-10)와 구체적 개선 방향 제시해줘.
```

### 취업 전략 관련

**프롬프트 19: 90일 취업 계획**
```
나의 90일 취업 준비 계획을 세워줘.

현재 상황:
- 교육 수료 예정일: [날짜]
- 보유 기술: [기술 목록]
- 포트폴리오 완성도: [퍼센트]
- 목표 직종: [직종]
- 목표 기업 유형: [대기업/중소/스타트업]

매주 구체적인 To-Do와 마일스톤을 포함해줘.
```

**프롬프트 20: 취업 이후 성장 계획**
```
신입 그래픽 디자이너로 취업했다고 가정하고,
첫 3개월 적응 계획을 세워줘.

직장 생활 초기:
- 빠르게 신뢰를 얻는 방법
- 업무 파악 순서
- 상사 및 동료와 관계 맺기
- 실수했을 때 대처 방법
- 퇴근 후 자기개발 방법
```

## 5. 포트폴리오 분석 프롬프트

### 종합 포트폴리오 분석

```
내 포트폴리오 PDF를 분석해줘. (파일 첨부)

분석 항목:
□ 전체 구성 및 흐름
□ 각 작업물의 완성도
□ 다양성 vs 전문성 균형
□ 레이아웃과 가독성
□ 디자이너의 실력과 성장 가능성
□ 취업 포트폴리오로서의 적합성

최종 평가: 몇 점짜리 포트폴리오인지 (100점 만점)
합격 가능한 기업 유형 추천
```

### 작업물별 심층 분석

```
이 브랜드 아이덴티티 작업물을 디자인 전문가 관점에서 분석해줘.

[이미지 첨부]

분석 항목:
1. 로고 디자인 완성도
2. 컬러 시스템의 일관성
3. 타이포그래피 적절성
4. 브랜드 스토리 전달력
5. 실제 적용 사례(목업)의 퀄리티
6. 면접에서 이 작업물 설명할 때 강조점
```

## 6. 자기소개서 프롬프트

### 첨삭 전문 프롬프트

```
너는 자기소개서 전문 컨설턴트야.
아래 자기소개서를 문단별로 첨삭해줘.

[자기소개서 전문]

첨삭 방식:
- 각 문단에 [★★★☆☆] 형식으로 점수 표시
- 문제 문장에 밑줄 표시: ~~나쁜 표현~~
- 개선 제안: → "개선된 표현"
- 전체 총평 및 수정 우선순위
```

### 키워드 매칭 프롬프트

```
채용공고와 내 자기소개서의 키워드 매칭률을 분석해줘.

채용공고 키워드: [키워드 목록]
내 자기소개서: [전문]

분석:
- 매칭된 키워드 (초록색 표시)
- 빠진 중요 키워드 (빨간색 표시)
- 매칭률 (퍼센트)
- 추가해야 할 키워드를 자연스럽게 넣는 방법
```

## 7. 프롬프트 개선하기

### 프롬프트 개선 사이클

```
1. 초기 프롬프트 작성
     ↓
2. 결과 확인
     ↓
3. 부족한 부분 식별
     ↓
4. "더 구체적으로 해줘" 또는 조건 추가
     ↓
5. 반복 개선
```

### 효과적인 피드백 표현

```
# 더 구체적으로 요청
"더 구체적인 예시를 포함해줘"
"2번 내용을 더 자세히 설명해줘"
"좀 더 감성적인 어조로 바꿔줘"

# 형식 조정
"표 형태로 정리해줘"
"더 짧게 요약해줘"
"불릿 포인트 대신 문장으로 써줘"

# 관점 변경
"면접관 관점에서 다시 평가해줘"
"부정적인 측면도 솔직하게 말해줘"
"처음부터 다시 다른 방식으로 작성해줘"
```
