## 1. 기업분석을 왜 하는가

### 기업분석의 3가지 목적

**목적 1: 지원 여부 결정**
- 이 회사가 내가 원하는 방향인가?
- 연봉, 복리후생, 성장 가능성 확인
- 기업 문화가 나와 맞는가?

**목적 2: 자기소개서 질 향상**
- 회사 맞춤형 지원동기 작성
- 채용공고 키워드와 회사 방향 연결
- "이 회사를 잘 안다"는 인상 전달

**목적 3: 면접 준비**
- 예상 면접 질문 예측
- 역질문 준비
- 포트폴리오 발표 방향 결정

### 기업분석 없이 면접 갔을 때의 위험

```
면접관: "저희 회사에 대해 아시는 게 있으신가요?"
준비 X: "잘 알려진 회사라서..."  ← 탈락 가능성 높음
준비 O: "최근 [신규 사업]을 시작하셨는데 
         디자인 언어가 변화하고 있더라고요..." ← 좋은 인상
```

## 2. Firecrawl로 기업 홈페이지 수집

### 수집해야 할 페이지 목록

| 페이지 | 수집 이유 |
|--------|-----------|
| 홈페이지 메인 | 회사의 최신 방향, 강조하는 것 |
| 회사 소개 (About) | 미션, 비전, 연혁 |
| 사업 분야 | 주요 제품/서비스 이해 |
| 채용 공고 | 요구하는 역량, 문화 힌트 |
| 뉴스/블로그 | 최신 동향, 방향성 |
| 디자인 팀 소개 (있다면) | 팀 규모, 역할 |

### Firecrawl 수집 스크립트

```python
from firecrawl import FirecrawlApp
from datetime import datetime
import json

app = FirecrawlApp(api_key="fc-xxxxxxxx")

def collect_company_info(company_name, company_url):
    """기업 정보 수집"""
    
    pages_to_collect = [
        {"url": company_url, "name": "메인"},
        {"url": f"{company_url}/about", "name": "회사소개"},
        {"url": f"{company_url}/careers", "name": "채용"},
        {"url": f"{company_url}/news", "name": "뉴스"},
    ]
    
    collected = {}
    
    for page in pages_to_collect:
        try:
            result = app.scrape_url(page["url"], params={'formats': ['markdown']})
            collected[page["name"]] = result.get('markdown', '')
            print(f"수집 완료: {page['name']}")
        except Exception as e:
            print(f"수집 실패 ({page['name']}): {e}")
    
    # 저장
    filename = f"기업분석_{company_name}_{datetime.now().strftime('%Y%m%d')}.json"
    with open(filename, 'w', encoding='utf-8') as f:
        json.dump(collected, f, ensure_ascii=False, indent=2)
    
    return collected

# 사용 예시
data = collect_company_info("카카오", "https://www.kakao.com")
```

### 수동 수집 방법 (Firecrawl 없이)

Firecrawl을 사용하기 어려운 경우:

1. 기업 홈페이지 접속
2. 주요 페이지의 텍스트를 복사 (Ctrl+A → Ctrl+C)
3. VSCode 또는 메모장에 붙여넣기
4. 파일로 저장 후 Claude에 첨부

## 3. Claude로 분석 요약

### 기업 분석 마스터 프롬프트

```
아래는 [회사명] 홈페이지에서 수집한 내용이야.
이 정보를 바탕으로 면접 준비용 기업 분석 보고서를 작성해줘.

[수집된 내용 붙여넣기]

보고서 구성:
1. 기업 개요 (3줄 요약)
2. 핵심 사업 분야 및 주요 제품/서비스
3. 기업 문화 및 핵심 가치
4. 디자인팀 역할 추론 (홈페이지 디자인 스타일 포함)
5. 최근 주요 이슈 또는 변화
6. 이 회사 지원 시 강조해야 할 포인트
7. 예상 면접 질문 5가지 + 모범 답변 방향
8. 역질문 3가지 (면접 마지막에 할 질문)
```

### 경쟁사 비교 분석

```
[회사 A 정보]와 [회사 B 정보]를 비교 분석해줘.

두 회사의 차이점:
1. 사업 방향성
2. 브랜드 아이덴티티 및 디자인 스타일
3. 기업 규모 및 성장 단계
4. 디자이너에게 제공하는 성장 기회
5. 연봉 및 처우 (공개 정보 기준)

최종: 내 커리어 목표를 고려했을 때 어디가 더 맞는지 추천해줘.
```

### 채용공고 + 기업분석 통합 분석

```
채용공고 내용과 회사 홈페이지 내용을 함께 분석해서
이 회사가 원하는 디자이너의 실제 모습을 파악해줘.

[채용공고 내용]
[회사 홈페이지 내용]

분석 포인트:
- 채용공고에 적힌 것 vs 실제로 하게 될 일
- 표면적 요구사항 vs 실제로 필요한 역량
- 신입에게 기대하는 실제 수준
- 문화적으로 잘 맞는 성격 유형
```

## 5. 면접 질문 연계

### 기업분석 → 면접 질문 예측

기업분석을 통해 다음과 같은 면접 질문을 예측할 수 있습니다:

**회사가 B2C 서비스 중심**이라면:
- "사용자 경험을 고려한 디자인 경험을 말해주세요"
- "소비자 입장에서 디자인을 바라본 경험이 있나요?"

**회사가 B2B 기업**이라면:
- "복잡한 정보를 시각적으로 정리한 경험이 있나요?"
- "다양한 이해관계자를 위한 디자인 경험을 말해주세요"

**스타트업**이라면:
- "빠른 속도로 작업하고 여러 분야를 함께 한 경험이 있나요?"
- "완성도와 속도 중 무엇을 더 중시하나요?"

**대기업**이라면:
- "팀워크와 협업 경험을 말해주세요"
- "프로세스를 따르는 것에 대해 어떻게 생각하나요?"

### 기업 특성별 면접 전략

```
스타트업 (10-50명 규모)
→ 강조: 멀티태스킹, 빠른 실행, 자기주도성
→ 포트폴리오: 완성도보다 다양한 경험

중견기업 (50-300명)
→ 강조: 팀 협업, 전문성, 성장 의지
→ 포트폴리오: 전문 분야 깊이

대기업 (300명 이상)
→ 강조: 프로세스 이해, 브랜드 이해도, 안정성
→ 포트폴리오: 브랜드 아이덴티티와의 일관성
```

## 6. 기업별 분석 실습

### 실습: 관심 기업 3개 분석

다음 단계를 따라 관심 기업 3개를 분석해보세요:

**Step 1**: 기업 목록 작성
```
관심 기업 목록:
1. 회사명: / 직무: / 마감일: 
2. 회사명: / 직무: / 마감일: 
3. 회사명: / 직무: / 마감일: 
```

**Step 2**: 각 기업 홈페이지 수집 (Firecrawl 또는 수동)

**Step 3**: Claude로 기업분석 보고서 생성

**Step 4**: 위 템플릿에 정리하여 파일 저장
```
저장 위치: cgd-ai-career-platform/기업분석DB/[회사명].md
```

**Step 5**: GitHub에 커밋
```bash
git add 기업분석DB/
git commit -m "docs: [회사명] 기업분석 보고서 추가"
```
