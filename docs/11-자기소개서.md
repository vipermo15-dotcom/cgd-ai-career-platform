## 1. 자기소개서 구성 이해

### 일반적인 항목 구성

대부분의 기업은 다음 4가지 항목을 요구합니다:

| 항목 | 내용 | 일반 분량 |
|------|------|-----------|
| 성장과정 | 인성과 가치관 형성 과정 | 500-700자 |
| 지원동기 | 이 회사에 지원한 이유 | 700-1000자 |
| 직무역량 | 업무 관련 강점과 경험 | 700-1000자 |
| 입사 후 포부 | 합격 시 기여 방향 | 500-700자 |

### 항목별 핵심 포인트

**성장과정**
- 어린 시절 전체를 서술하지 않기
- 디자이너가 되려고 한 결정적 계기에 집중
- 가치관과 성격의 형성에 집중

**지원동기**
- "좋은 회사라서"는 절대 금지
- 구체적인 사업·제품·서비스에 대한 언급 필요
- 회사의 미션/비전과 본인의 연결고리

**직무역량**
- 수치로 증명 가능한 경험 포함
- STAR 기법 활용 (상황-과제-행동-결과)
- 해당 직무와 직접 연결되는 경험

**입사 후 포부**
- "열심히 하겠습니다" 금지
- 구체적인 1년/3년/5년 계획
- 회사의 사업 방향과 연계

## 2. AI로 초안 작성하기

### 사전 준비: 나의 재료 정리

AI에게 좋은 자기소개서를 요청하려면 먼저 재료를 정리해야 합니다.

**자기소개서 재료 수집 양식**:

```markdown
## 내 정보 정리

### 기본 정보
- 이름: 
- 지원 직무: 
- 지원 회사: 
- 제출 마감: 

### 디자인과 연결된 경험
1. 디자인을 처음 접한 계기:
2. 가장 기억에 남는 작업 경험:
3. 어려움을 극복한 경험:
4. 팀 협업 경험:
5. 성장을 보여주는 에피소드:

### 보유 역량
- 기술 역량: Adobe CC (Photoshop, Illustrator, InDesign 등)
- 소프트 스킬: 커뮤니케이션, 분석력, 창의성 등
- 자격증:

### 지원 회사 분석
- 회사의 주요 사업:
- 내가 관심 있는 제품/서비스:
- 지원동기 (솔직하게):
```

### 성장과정 AI 작성 프롬프트

```
너는 자기소개서 전문 컨설턴트야.
아래 재료를 바탕으로 성장과정 600자를 작성해줘.

핵심 에피소드:
- 고등학교 때 포스터 대회에서 처음 디자인 프로그램을 접함
- 결과물이 친구들에게 인정받았을 때 보람을 느낌
- 그 이후 독학으로 Adobe CC를 배우기 시작

작성 방향:
- 디자이너가 되고 싶다는 결심이 언제, 왜 생겼는지
- 과정에서 배운 가치관 (완성도에 대한 집착, 사용자 관점 등)
- 역경을 극복한 성장 스토리
- 첫 문장부터 강렬하게 시작
- 상투적 표현 금지 (어릴 때부터, 항상 꿈꿔왔습니다 등)
```

### 지원동기 AI 작성 프롬프트

```
나는 [회사명]의 [직무]에 지원하려고 해.
아래 정보를 바탕으로 지원동기 900자를 작성해줘.

회사 정보:
[Firecrawl로 수집한 회사 홈페이지 내용 붙여넣기]

지원 이유:
1. 이 회사의 [특정 제품/서비스]가 내 삶에 영향을 준 경험
2. 이 회사의 디자인 스타일이 내 추구하는 방향과 일치
3. 성장 가능성 (업계 내 입지, 디자인 투자)

작성 조건:
- 첫 문장: 회사의 특정 제품/서비스 언급으로 시작
- 회사 연구를 충분히 했다는 것이 드러나도록
- 나의 역량이 회사에 어떻게 기여할지 포함
- "귀사" 등 딱딱한 표현보다 자연스러운 어조
```

## 3. 채용공고 키워드 분석

### 키워드 추출 방법

**Step 1**: 채용공고 전문 복사

**Step 2**: Claude에게 키워드 추출 요청

```
이 채용공고에서 자기소개서에 반드시 포함해야 할
키워드를 중요도 순으로 추출해줘.

[채용공고 전문]

추출 방법:
- 필수 자격요건 → 반드시 언급
- 우대사항 → 해당되면 적극 언급
- 업무 내용에 반복되는 단어 → 나의 언어로 재표현
- 회사 가치관 키워드 → 자기소개서 곳곳에 녹이기
```

**Step 3**: 키워드를 자기소개서에 자연스럽게 포함

```
채용공고 키워드: 창의적 사고, 협업, Adobe CC 숙련

자기소개서에 자연스럽게 포함하는 방법:
× "저는 창의적이고 협업을 잘 합니다"
✓ "팀 프로젝트에서 서로 다른 의견을 시각적으로 통합하는 과정에서..."
```

### 키워드 매핑 테이블 만들기

```markdown
| 채용공고 키워드 | 내 경험/역량 | 자기소개서 포함 위치 |
|----------------|-------------|---------------------|
| Adobe CC 숙련 | 1년 이상 실무 수준 사용 | 직무역량 |
| 커뮤니케이션 | 그룹 프로젝트 5회 | 성장과정/직무역량 |
| 마감 준수 | 교육 과제 모두 기한 내 완료 | 직무역량 |
| 창의적 사고 | OO 공모전 수상 경험 | 성장과정 |
```

## 4. 첨삭 요청 프롬프트

### 1차 첨삭: 문장 수준

```
아래 자기소개서를 문장 수준에서 첨삭해줘.

[자기소개서 전문]

첨삭 기준:
1. 상투적 표현 찾기 → 구체적 대안 제시
2. 너무 길거나 복잡한 문장 → 간결하게 수정
3. 수동태 → 능동태로 변환
4. 중복 표현 → 삭제 또는 합치기
5. 각 문장에 수정 여부와 이유를 표시해줘
```

### 2차 첨삭: 구조 수준

```
아래 자기소개서의 전체 구조를 분석해줘.

[자기소개서 전문]

분석 항목:
1. 각 단락의 핵심 메시지가 명확한가?
2. 단락 간의 흐름이 자연스러운가?
3. 지원동기가 회사 분석과 연결되어 있는가?
4. 마무리가 기억에 남을 만큼 강렬한가?
5. 전체적으로 "이 사람을 채용하고 싶다"는 느낌이 드는가?
```

### 3차 첨삭: 채용담당자 관점

```
너는 [회사명]의 채용 담당자야.
아래 자기소개서를 보고 합격/불합격 결정과 이유를 알려줘.

[자기소개서 전문]

평가 항목:
- 직무 적합성 (0-30점)
- 경험의 구체성 (0-30점)
- 회사 이해도 (0-20점)
- 문장 표현력 (0-20점)
- 총점과 합격 여부
- 불합격 시 개선해야 할 가장 중요한 한 가지
```

## 5. 실제 예시 (디자인 직군)

### 예시 1: 그래픽 디자이너 지원동기 (1000자)

```
처음 '아모레퍼시픽' 브랜드 패키지를 손에 쥐었을 때, 
상자 하나가 이렇게 많은 이야기를 담을 수 있다는 것을 알았습니다.
그 순간이 제가 패키지 디자인에 매료된 시작이었습니다.

서울시기술교육원 컴퓨터그래픽디자인과에서 1년간 공부하며
디자인은 단순한 꾸밈이 아닌 브랜드의 철학을 담는 언어임을 배웠습니다.
특히 편집 디자인 수업에서 단 한 줄의 여백이 
독자의 독서 경험을 얼마나 바꾸는지를 체험한 후,
공간과 여백을 통해 이야기하는 아모레퍼시픽의 디자인 철학에 
더욱 공감하게 되었습니다.

수업 외에도 3개의 브랜딩 프로젝트를 직접 기획하고 제작했습니다.
카페 브랜드 아이덴티티 프로젝트에서는 
로고부터 패키지, 유니폼까지 통합적 브랜드 경험을 설계하며
디자인의 일관성이 브랜드 신뢰를 만든다는 것을 경험했습니다.

아모레퍼시픽은 K-뷰티를 넘어 글로벌 라이프스타일 브랜드로
성장하고 있습니다. 저는 한국적 아름다움과 글로벌 트렌드를
섬세하게 연결하는 이 회사에서, 소비자의 일상에 스며드는
디자인을 함께 만들고 싶습니다.

신입이지만 배움의 속도와 완성도에 대한 집착으로
빠르게 기여하는 디자이너가 되겠습니다.
```

### 예시 2: UI/UX 디자이너 직무역량 (900자)

```
사용자가 불편함을 느끼지 않는 것이 최고의 디자인이라 생각합니다.

교육과정에서 카카오 라이더스 앱을 UX 리서치 대상으로 선택하여
실제 배달 기사 3명을 인터뷰하고 사용 패턴을 관찰했습니다.
"장갑을 끼고도 조작해야 한다"는 인사이트를 발견하고,
터치 영역을 기존 대비 40% 확장한 재설계안을 제안했습니다.
이 경험으로 디자인은 사용자의 맥락에서 시작해야 함을 배웠습니다.

Figma를 활용한 프로토타입 제작에서는
컴포넌트 시스템을 구축하여 팀원들과 협업하는 방식으로 진행했습니다.
3명의 팀원이 같은 디자인 언어를 사용하면서
수정 시간을 기존 대비 60% 단축하는 경험을 했습니다.
이를 통해 디자인 시스템의 중요성을 체감했습니다.

보유 스킬로는 Figma, Adobe XD, Photoshop, Illustrator가 있으며
사용자 인터뷰, Affinity Map, 와이어프레임, 프로토타이핑까지
전체 UX 프로세스를 경험했습니다.

[회사명]의 앱에서 더 나은 사용자 경험을 만드는
데이터 기반의 디자인을 실현하고 싶습니다.
```

### 예시 3: 성장과정 (650자)

```
처음 포토샵을 열었던 날, 모니터 앞에서 6시간을 보냈습니다.
간단한 배경 지우기 하나에 그토록 오래 걸린 것이 창피했지만,
결국 해냈을 때의 성취감이 저를 더 깊이 빠져들게 만들었습니다.

고등학교 3학년 때, 학교 축제 포스터 제작을 맡았습니다.
처음 만든 포스터를 선생님께 보여드렸을 때 돌아온 말은
"글씨가 너무 많아서 한눈에 안 들어온다"였습니다.
그 한마디에 3번을 다시 만들었고, 최종 포스터는
축제장 입구에 크게 인쇄되어 붙었습니다.
처음으로 많은 사람들이 내 디자인 앞에서 사진을 찍는 것을 봤습니다.

이 경험으로 두 가지를 배웠습니다.
첫째, 디자인은 보여주는 것이 아니라 전달하는 것이다.
둘째, 피드백을 두려워하지 않는 사람이 더 빨리 성장한다.

서울시기술교육원에서 체계적으로 배우면서
기술적 역량뿐만 아니라 의뢰인의 요구를 이해하고
그것을 시각적으로 해결하는 디자인 사고력을 키워왔습니다.
앞으로도 성장을 두려워하지 않는 디자이너가 되겠습니다.
```

## 6. 자기소개서 최종 점검

### 제출 전 체크리스트

**내용 점검**
- [ ] 각 항목이 요구 분량을 충족하는가? (±10% 이내)
- [ ] 상투적 표현이 없는가? (항상, 어릴 때부터, 열심히 등)
- [ ] 구체적인 에피소드가 포함되어 있는가?
- [ ] 회사명이 다른 회사 이름으로 잘못 기재되지 않았는가?
- [ ] 숫자와 날짜가 정확한가?

**언어 점검**
- [ ] 오탈자 없음
- [ ] 맞춤법 오류 없음 (네이버 맞춤법 검사기 활용)
- [ ] 문장 부호 (쉼표, 마침표) 일관성
- [ ] 존댓말 일관성 (합니다 체 또는 입니다 체 통일)

**최종 AI 점검**

```
이 자기소개서를 제출하기 전 최종 점검해줘.

[자기소개서 전문]

점검 항목:
1. 오탈자·맞춤법 오류
2. 상투적 표현
3. 내용의 일관성
4. 전체 인상 (1-10점)
5. 한 가지만 수정할 수 있다면?
```
