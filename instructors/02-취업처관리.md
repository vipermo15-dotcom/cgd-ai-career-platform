# 취업처 관리 가이드 - 교수자용

## 개요
학생들의 취업 목표 기업을 체계적으로 관리하고, 취업처 파이프라인을 운영하는 방법입니다.

---

## 1. 취업처 파이프라인 개념

취업처 파이프라인은 학생 한 명의 취업 활동을 단계별로 추적하는 시스템입니다.

```
파이프라인 단계:

[관심] → [서류준비] → [지원완료] → [서류합격] → [면접예정] → [면접완료] → [합격/불합격]
```

### 각 단계 설명

| 단계 | 의미 | 교수자 할 일 |
|------|------|------------|
| 관심 | 지원 고려 중 | 기업 정보 제공 |
| 서류준비 | 서류 작성 중 | 포트폴리오/자소서 피드백 |
| 지원완료 | 실제 지원 완료 | 결과 대기, 연락 대기 |
| 서류합격 | 서류 통과 | 면접 준비 코칭 |
| 면접예정 | 면접 일정 잡힘 | 모의 면접 진행 |
| 면접완료 | 면접 응시 | 피드백 수렴, 결과 대기 |
| 합격 | 최종 합격 | 축하 + 취업 DB 업데이트 |
| 불합격 | 탈락 | 피드백 분석, 재도전 전략 |

---

## 2. CGD 플랫폼 업체 파이프라인 활용

### 2.1 학생별 파이프라인 조회

1. CGD 플랫폼 > 관리자 메뉴 > 취업 현황
2. 학생 선택
3. 파이프라인 뷰에서 현재 단계 확인
4. 각 기업별 진행 상황 확인

### 2.2 파이프라인 업데이트 방법

**교수자 직접 업데이트**
```
1. 학생으로부터 결과 보고 받음
2. 플랫폼 > 학생 관리 > 해당 학생 클릭
3. 지원 현황 탭 선택
4. 해당 기업 단계 변경
5. 메모 추가 (면접 분위기, 피드백 내용 등)
```

**학생 자기 업데이트 (권장)**
```
1. 학생이 직접 로그인
2. 내 취업 현황 메뉴
3. 해당 기업 단계 업데이트
4. 교수자에게 자동 알림 발송
```

## 3. 채용 의뢰 프로세스

### 3.1 기업에서 채용 의뢰가 왔을 때

```
Step 1: 의뢰 접수
- 기업 담당자 이메일 또는 전화로 의뢰
- 의뢰서 양식 요청 (직군, 분야, 우대 조건)

Step 2: 학생 매칭
- CGD 플랫폼 > 취업 현황 > 학생 필터
- 해당 직군 + 준비 상태 "서류준비" 이상인 학생 조회
- 최대 3명 추천 (학생 동의 필요)

Step 3: 학생 안내
- 개별 면담으로 의뢰 내용 공유
- 관심 여부 확인
- 포트폴리오 최신화 안내

Step 4: 연결
- 학생 포트폴리오 + 이력서 기업에 전달
- 기업-학생 직접 연락 연결

Step 5: 추적
- 진행 상황 파이프라인에 업데이트
- 결과 확인 및 기록
```

## 4. 취업률 관리

### 4.1 취업률 계산 기준

```
취업률 = (취업 인원 / 졸업 인원) × 100

포함 기준:
- 정규직 취업
- 계약직 취업 (6개월 이상)
- 자영업 창업

제외 기준:
- 아르바이트
- 진학
- 군입대
```

### 4.2 월별 취업 현황 리포트

```markdown
# 2026년 1월 취업 현황 리포트

## 요약
- 총 학생: 30명
- 취업 완료: 8명 (26.7%)
- 지원 중: 18명
- 미시작: 4명

## 직군별 현황
| 직군 | 취업 | 지원중 |
|------|------|--------|
| 그래픽 디자이너 | 4명 | 10명 |
| UI/UX | 2명 | 5명 |
| 영상 편집 | 1명 | 2명 |
| 기타 | 1명 | 1명 |

## 이달 합격 기업
1. [학생명] → [기업명] (그래픽 디자이너)
2. [학생명] → [기업명] (UI/UX)

## 주요 이슈
- 포트폴리오 완성도 부족 학생 5명 → 추가 지원 필요
- 기업 B 면접 피드백: 실무 능력 부족 → 툴 교육 강화 제안
```
