# 교수자 운영 매뉴얼 - CGD AI Career Platform

## 매뉴얼 개요

이 문서는 CGD AI Career Platform을 운영하는 교수자를 위한 전체 가이드입니다.
플랫폼 접속부터 학기 운영, 학생 관리까지 모든 내용을 담고 있습니다.

**플랫폼 URL**: https://cgdplatform-pm7dnqip.manus.space  
**GitHub 저장소**: github.com/cgd-team/cgd-ai-career-platform  
**최종 수정**: 2026년 1월

## 1. 플랫폼 전체 운영 가이드

### 1.1 시스템 구성 전체 그림

```
CGD AI Career Platform
├── 학생 facing
│   ├── 취업지원 에이전트 (Claude 기반)
│   ├── 포트폴리오 첨삭 에이전트
│   ├── 기업분석 에이전트 (Firecrawl)
│   ├── 면접 준비 에이전트
│   └── 채용공고 조회
│
├── 교수자 facing
│   ├── 학생 관리 대시보드
│   ├── 취업 현황 트래킹
│   ├── 기업 DB 관리
│   └── 교육 자료 관리
│
└── 자동화 시스템
    ├── GitHub Actions (채용공고 자동 수집)
    ├── Firecrawl (기업정보 모니터링)
    └── Claude API (분석 자동화)
```

### 1.2 교수자 계정 관리

**관리자 계정**
- URL: cgdplatform-pm7dnqip.manus.space/admin
- 계정: 초기 설정 시 발급된 관리자 계정 사용
- 비밀번호 정책: 90일마다 변경 권장

**학생 계정 발급**
1. 관리자 로그인
2. 학생 관리 → 신규 학생 등록
3. 이름, 이메일, 학번 입력
4. 임시 비밀번호 자동 발급 (이메일 발송)
5. 첫 로그인 시 비밀번호 변경 안내

## 2. 학기 시작 준비사항

### 2.1 학기 시작 4주 전 (체크리스트)

```
[ ] GitHub 저장소 최신화
    - 교육 자료 버전 업데이트
    - 채용공고 데이터 최신화
    - 예제 파일 검토

[ ] CGD 플랫폼 설정
    - 학기 시작일 설정
    - 새 학생 계정 일괄 생성
    - 에이전트 시스템 프롬프트 업데이트

[ ] 기업 DB 업데이트
    - 협력 기업 정보 최신화
    - Firecrawl로 채용 페이지 재수집
    - 새 기업 추가 (최소 10곳)

[ ] API 키 확인
    - Firecrawl API 크레딧 확인 (최소 1000 크레딧)
    - Anthropic API 잔액 확인
    - GitHub Actions 워크플로우 테스트
```

### 2.2 학기 시작 1주 전

```
[ ] 첫 수업 준비
    - Week01 자료 최종 확인
    - 학생 GitHub 계정 가이드 준비
    - 실습 환경 테스트 (Node.js, Python)

[ ] 학생 오리엔테이션 자료
    - 플랫폼 사용법 안내 PPT
    - 계정 발급 정보 이메일 발송
    - 수업 일정 공지

[ ] 기술 점검
    - 교실 PC GitHub 접속 확인
    - 프로젝터 연결 확인
    - 인터넷 연결 속도 확인
```

## 3. 주차별 교수자 할 일

### Week 01 - Git/GitHub 기초
```
수업 전:
[ ] GitHub 계정 생성 방법 스크린샷 준비
[ ] Git 설치 파일 USB에 준비 (인터넷 느린 경우 대비)
[ ] 학생별 GitHub 계정 미리 확인 (이전 경험 여부)

수업 중:
[ ] 개인별 Git 설치 확인
[ ] PAT(Personal Access Token) 발급 안내
[ ] 저장소 생성 → 커밋 → push 실습 감독

수업 후:
[ ] 미션 제출 현황 확인
[ ] 미제출 학생 개별 연락
[ ] 다음 주 Markdown 자료 확인
```

### Week 02~05 - 도구 셋업 기간
```
매주 반복:
[ ] 지난 주 미션 채점 (마감 후 48시간 이내)
[ ] 개별 피드백 댓글 작성 (GitHub PR 또는 플랫폼)
[ ] 어려움 겪는 학생 파악 → 추가 지원
[ ] 수업 자료 업데이트 (필요시)
```

### Week 06~10 - 포트폴리오 제작 기간
```
매주 추가 업무:
[ ] 포트폴리오 중간 점검 (Week07, Week09)
[ ] 개별 포트폴리오 피드백 제공
[ ] Behance 업로드 확인 (Week08 이후)
[ ] CGD 플랫폼 포트폴리오 첨삭 에이전트 활용 안내
```

### Week 11~15 - 취업 준비 기간
```
매주 추가 업무:
[ ] 기업 DB 최신화 (새 채용공고 추가)
[ ] 학생별 지원 현황 1:1 면담
[ ] 모의 면접 스케줄 조율 (Week12)
[ ] 협력 기업 연락 (취업 의뢰 확인)
```

### Week 16~20 - 최종 점검 기간
```
매주 추가 업무:
[ ] 자기소개서 최종 검토 (교수자 직접 확인)
[ ] 졸업 발표 스케줄 확정
[ ] 취업처 파이프라인 최종 점검
[ ] 취업률 집계 준비
```

## 4. 교수자 도구 활용 가이드

### 4.1 Claude Desktop + MCP 활용

**교수자용 활용 사례**
```
1. 학생 포트폴리오 일괄 피드백
   - GitHub MCP로 저장소 일괄 읽기
   - Claude가 각 저장소 README 분석
   - 개별 피드백 자동 생성

2. 주간 취업 현황 리포트 자동 생성
   - 플랫폼 데이터 → Claude 분석 → Markdown 보고서

3. 기업 채용 트렌드 분석
   - Firecrawl로 주요 기업 채용 페이지 수집
   - Claude로 직군별 요구 스킬 변화 분석
```

### 4.2 GitHub Actions 자동화

교수자가 직접 관리해야 할 워크플로우:

```
.github/workflows/
├── weekly-jobs.yml        - 주간 채용공고 자동 수집
├── monthly-report.yml     - 월간 취업 현황 리포트
├── company-monitor.yml    - 협력 기업 채용 변동 감지
└── update-toc.yml         - 문서 목차 자동 업데이트
```

워크플로우 실행 확인:
1. GitHub → Actions 탭
2. 각 워크플로우 최근 실행 결과 확인
3. 실패 시 오류 로그 확인 → 수정

## 5. 응급 상황 대처

### API 키 초과 시
```
Firecrawl 크레딧 소진:
→ firecrawl.dev에서 크레딧 충전
→ 또는 수동 수집으로 대체 (2~3일)

Anthropic API 한도 초과:
→ usage.anthropic.com에서 확인
→ 플랜 업그레이드 또는 월 초까지 대기
```

### 플랫폼 오류 시
```
학생이 로그인 안 됨:
→ 관리자 페이지에서 계정 상태 확인
→ 비밀번호 초기화

에이전트가 응답 안 함:
→ API 키 상태 확인
→ 에이전트 재시작 (관리자 설정)
```
