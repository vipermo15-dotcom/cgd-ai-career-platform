# Week 01 예제 - Git/GitHub 실제 사용 예시

## 예제 1: 좋은 커밋 메시지 vs 나쁜 커밋 메시지

### 나쁜 예시 (절대 이렇게 하지 마세요)
```bash
git commit -m "수정"
git commit -m "1"
git commit -m "aaaa"
git commit -m "작업함"
```

### 좋은 예시 (이렇게 작성하세요)
```bash
git commit -m "docs: 자기소개 README 추가"
git commit -m "feat: 포트폴리오 목차 구성"
git commit -m "fix: 이메일 주소 오타 수정"
git commit -m "design: 표지 이미지 업데이트"
```

### 커밋 메시지 유형 접두사
| 접두사 | 의미 | 예시 |
|--------|------|------|
| `feat:` | 새 기능/내용 추가 | `feat: 프로젝트1 포트폴리오 추가` |
| `fix:` | 오류 수정 | `fix: 전화번호 오타 수정` |
| `docs:` | 문서 작성/수정 | `docs: 자기소개 업데이트` |
| `design:` | 디자인 변경 | `design: 색상 팔레트 변경` |
| `init:` | 초기 설정 | `init: 저장소 초기화` |

## 예제 2: 실제 포트폴리오 저장소 구조

취업을 위한 포트폴리오 저장소는 이렇게 구성합니다:

```
portfolio-2026/
├── README.md                    ← 메인 소개 페이지
├── .gitignore                   ← 업로드 제외 파일 목록
│
├── projects/                    ← 프로젝트 폴더
│   ├── 01-brand-identity/       ← 브랜드 아이덴티티 프로젝트
│   │   ├── README.md            ← 프로젝트 설명
│   │   ├── concept.md           ← 기획 의도
│   │   └── images/              ← 결과물 이미지
│   ├── 02-packaging-design/
│   └── 03-motion-graphic/
│
├── skills/                      ← 보유 스킬 정리
│   └── tools.md
│
├── certificates/                ← 자격증 및 수상
│   └── README.md
│
└── contact/                     ← 연락처
    └── README.md
```

## 예제 4: GitHub에서 실제로 보이는 화면 설명

```
github.com/minjun-kim/portfolio-2026
├── 📄 README.md          ← 자동으로 메인 화면에 표시됨
├── 📁 projects/          ← 폴더 클릭하면 내용 확인 가능
├── 📁 skills/
└── Commits: 15           ← 커밋 횟수 (많을수록 성실함 어필)
    Contributors: 1       ← 기여자 수
    Last commit: 2일 전   ← 최근 활동 일자
```

## 예제 5: 실제 취업 시 GitHub 활용법

### 채용 담당자가 GitHub에서 확인하는 것

1. **프로필 완성도**: 사진, 소개, 연락처 있는지
2. **저장소 관리**: README가 잘 작성되어 있는지
3. **커밋 활동**: 꾸준히 작업했는지 (잔디 그래프)
4. **코드/파일 품질**: 폴더 구조가 깔끔한지
5. **포트폴리오 링크**: 실제 작품 보여주는지

### 이력서에 GitHub 링크 넣는 법

```
포트폴리오: github.com/minjun-kim/portfolio-2026
```

또는 자기소개서에:
```
"저의 모든 디자인 작업물은 GitHub(github.com/minjun-kim)에서 
버전별로 관리하고 있으며, 제작 과정의 변화를 투명하게 공유합니다."
```
