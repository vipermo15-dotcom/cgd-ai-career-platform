# Week 01 실습 - Git/GitHub 기초

## 이번 주 실습 목표
Git과 GitHub를 설치하고 첫 번째 저장소를 만들어 코드를 업로드합니다.

---

## 실습 1: Git 설치 확인

```bash
# Git 설치 확인
git --version
# 출력 예: git version 2.43.0
```

설치가 안 되어 있다면:
- Mac: `brew install git`
- Windows: https://git-scm.com/download/win 에서 다운로드

---

## 실습 2: Git 초기 설정

```bash
# 사용자 이름 설정 (GitHub 아이디와 동일하게)
git config --global user.name "홍길동"

# 이메일 설정 (GitHub 가입 이메일)
git config --global user.email "gildong@email.com"

# 설정 확인
git config --list
```

---

## 실습 3: 첫 번째 저장소 만들기

### 로컬에서 시작하기

```bash
# 폴더 만들기
mkdir my-portfolio
cd my-portfolio

# Git 저장소 초기화
git init

# README 파일 만들기
echo "# 내 포트폴리오" > README.md

# 파일 추가
git add README.md

# 첫 커밋
git commit -m "첫 번째 커밋: README 추가"
```

---

## 실습 4: GitHub에 업로드

### GitHub 저장소 생성
1. github.com 접속 → 로그인
2. 오른쪽 상단 `+` → `New repository` 클릭
3. Repository name: `my-portfolio` 입력
4. `Create repository` 클릭

### 로컬과 GitHub 연결

```bash
# 원격 저장소 연결
git remote add origin https://github.com/내아이디/my-portfolio.git

# 메인 브랜치로 변경
git branch -M main

# 업로드 (push)
git push -u origin main
```

---

## 실습 5: 파일 수정 후 다시 업로드

```bash
# 파일 내용 추가
echo "안녕하세요! 그래픽 디자이너 지망생입니다." >> README.md

# 변경 사항 확인
git status
git diff README.md

# 변경 사항 추가 및 커밋
git add README.md
git commit -m "README에 자기소개 추가"

# GitHub에 반영
git push
```

## 자주 하는 실수

| 실수 | 해결 방법 |
|------|----------|
| push 시 인증 오류 | GitHub > Settings > Developer settings > Tokens에서 PAT 생성 |
| `not a git repository` 오류 | `git init` 실행 여부 확인 |
| 파일이 보이지 않음 | `git add` 후 `git commit` 순서 확인 |
