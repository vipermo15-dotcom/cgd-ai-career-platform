# Week 02 예제 - Markdown 실제 활용 예시

## 예제 1: 포트폴리오 프로젝트 설명 Markdown

실제 포트폴리오에 들어갈 프로젝트 설명 페이지 예시입니다.

```markdown
# 한국茶 브랜드 아이덴티티 리디자인

> "전통의 가치를 현대적 감성으로"

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 클라이언트 | 한국茶 (가상 브랜드) |
| 프로젝트 기간 | 2025년 9월 ~ 11월 (3개월) |
| 담당 역할 | 브랜드 아이덴티티 전체 |
| 사용 도구 | Illustrator, Photoshop, InDesign |

## 프로젝트 배경

한국의 전통 차 문화를 MZ세대에게 어필하기 위한 브랜드 리디자인 프로젝트입니다.
기존 로고의 정체성은 유지하면서 현대적 감성을 더하는 것이 핵심 과제였습니다.

## 디자인 프로세스

1. **리서치** - 경쟁 브랜드 분석, 타겟 설문조사
2. **컨셉 개발** - 무드보드, 키워드 도출
3. **로고 시안** - 3가지 방향성 제시
4. **선정 및 정교화** - 클라이언트 피드백 반영
5. **어플리케이션** - 명함, 패키지, 디지털 적용

## 결과물

### 로고
![로고](images/logo-final.jpg)

### 컬러 시스템
- Primary: #2C5F2E (숲 초록)
- Secondary: #D4A017 (차 황금)
- Background: #F5F0E8 (한지 아이보리)

### 폰트
- 제목: Noto Serif KR (한국 서체 감성)
- 본문: Noto Sans KR (가독성)

## 배운 점

이 프로젝트를 통해 전통과 현대의 조화가 단순히 스타일의 문제가 아니라
브랜드의 스토리텔링과 깊이 연결되어 있다는 것을 배웠습니다.

---

```

## 예제 2: 채용공고 분석 Markdown

```markdown
# 카카오 UX 디자이너 채용공고 분석

**분석일**: 2026년 1월 15일  
**원본 링크**: https://careers.kakao.com/jobs/12345

## 공고 핵심 정보

| 항목 | 내용 |
|------|------|
| 직군 | UX 디자이너 (신입/경력) |
| 근무지 | 판교 |
| 마감일 | 2026년 2월 28일 |
| 예상 연봉 | 3,000~4,500만원 |

## 필수 자격 요건

- [ ] Figma 능숙 사용자
- [ ] UX 리서치 경험
- [ ] 사용성 테스트 수행 가능
- [ ] 포트폴리오 보유

## 나의 준비 현황

| 요건 | 나의 현황 | 준비 필요사항 |
|------|----------|--------------|
| Figma | ★★★☆☆ | 더 연습 필요 |
| UX 리서치 | ★★☆☆☆ | 프로젝트 추가 |
| 사용성 테스트 | ★☆☆☆☆ | 학습 필요 |
| 포트폴리오 | 제작 중 | 2월까지 완성 |

## 준비 계획

1. **이번 주**: Figma 기초 강의 완료
2. **2주 후**: 개인 UX 프로젝트 시작
3. **1달 후**: 포트폴리오 초안 완성
4. **마감 전**: 최종 지원
```

## 예제 3: 기업 조사 보고서 Markdown

```markdown
# 네이버 기업 분석 보고서

> CGD 취업지원 플랫폼 Firecrawl + Claude 분석 결과

## 기업 기본 정보

| 항목 | 내용 |
|------|------|
| 기업명 | 네이버㈜ |
| 설립 | 1999년 |
| 직원수 | 약 4,000명 (2025 기준) |
| 주요 서비스 | 검색, 웹툰, 쇼핑, 클라우드 |
| 채용 페이지 | recruit.navercorp.com |

## 디자인 직군 현황

### 채용 트렌드 (최근 1년)
- UX/UI 디자이너 ████████ 40%
- 브랜드 디자이너 ████ 20%
- 모션/영상 ████ 20%
- 기타 ████ 20%

### 우대 스킬
1. Figma 고급 사용자
2. 데이터 기반 디자인 경험
3. Design System 구축 경험
4. A/B 테스트 경험

## 인재상 키워드

> "기술과 디자인의 경계를 넘는 크리에이터"

- **도전**: 새로운 것을 두려워하지 않는 자세
- **협업**: 다양한 직군과 원활한 소통
- **성과**: 데이터로 증명하는 디자인

## 나의 지원 전략

### 포트폴리오 포인트
- [x] 사용자 중심 설계 프로세스 강조
- [x] 데이터/근거 기반 디자인 결정 과정 포함
- [ ] 모바일 UX 프로젝트 추가 필요

### 자기소개서 키워드
사용자, 데이터, 협업, 개선, 임팩트, 기술, 창의

---
*분석 도구: Firecrawl + Claude API | 분석일: 2026-01-15*
```
